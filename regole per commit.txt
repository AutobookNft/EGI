Da ora in poi voglio che tu scriva per me i messaggi dei commit Git seguendo queste regole:

1. Ogni commit deve iniziare con un TAG maiuscolo tra parentesi quadre che indichi il tipo di azione:
   - [FEAT] = nuova feature o funzionalità
   - [FIX] = bug risolto
   - [REFACTOR] = refactoring del codice senza cambiare logica
   - [DOC] = documentazione aggiunta o aggiornata
   - [TEST] = aggiunta o modifica di test
   - [CHORE] = attività di manutenzione o setup (config, dipendenze, script, ecc.)

2. Dopo il tag, scrivi una frase chiara e concisa che descriva COSA è stato fatto, al presente.
   Esempio: `[FIX] Corretto bug di rendering nelle egi-card su mobile`.

3. Se il commit è più complesso, aggiungi un elenco puntato (max 4–5 punti) che spieghi i dettagli principali:
   - cosa hai modificato
   - perché lo hai fatto (breve)
   - eventuali effetti collaterali / note

4. Usa un linguaggio tecnico ma leggibile: niente frasi vaghe come "fix bug" o "update stuff".

5. Mantieni lo stile uniforme in tutti i commit.

Alla fine, il messaggio deve essere sempre:
[TAG] descrizione breve
+ elenco puntato (se necessario)

Questo formato deve essere rispettato SEMPRE.
